# Magic-UI

在前面页面渲染open in new window中提到的 UI 渲染器，就是包含在 Magic-UI 中的渲染器组件。

魔方的设计是希望发布的页面支持多个前端框架，即各个业务方可以根据自己熟悉的语言来开发组件、发布页面。也可以通过 实现一个 runtimeopen in new window 的方式，来实现一个自己的 Magic-UI。

所以魔方的设计中，针对每个前端框架，都需要有一个对应的 Magic-UI 来承担渲染器职责。同时，也需要一个使用和 Magic-UI 相同前端框架的 runtime 来 Magic-UI 和业务组件的，具体 runtime 概念，可以参考页面发布open in new window。

Magic-UI 在魔方设计中，承担的是业务逻辑无关的，基础组件渲染的功能。一切和业务相关的逻辑，都应该在 runtimeopen in new window 中实现。这样 Magic-UI 就能保持其通用性。

我们以项目代码中提供的 vue3 版本的 Magic-UI 作为示例介绍其中包含的内容。

# 基础组件

在 Magic-UI vue3 中，我们提供了几个基础组件，可以在项目源码中找到对应内容。

  • page 魔方的页面基础
  • container 魔方的容器渲染器
  • Component.vue 魔方的组件渲染器
  • button/text 基础组件示例

其中 page/container/Component 是 UI 的基础，是每个框架的 UI 都应该实现的。

button/text 其实就是一个组件开发的示例，具体组件开发相关规范可以参考组件开发open in new window。
